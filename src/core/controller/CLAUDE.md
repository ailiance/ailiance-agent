# Controller — gRPC dispatch hub

Routes webview/CLI RPCs to the agent core. `Controller` (`index.ts`) owns the
active `Task`, `StateManager`, and workspace manager; every UI action lands here
as a protobus call before reaching `core/task/`.

## Request flow

1. Host receives `grpc_request` → `handleGrpcRequest` (`grpc-handler.ts`).
2. `getHandler(service, method)` looks up `serviceHandlers` (generated, in
   `@generated/hosts/vscode/protobus-services`) — NOT hand-wired.
3. Handler is called as `handler(controller, message[, responseStream, requestId])`.
4. Unary → returns one proto message. Streaming (`is_streaming`) → pushes through
   `StreamingResponseHandler`; the stream stays open until client disconnect.
5. `withRecordingMiddleware` mirrors every request/response into `grpc-recorder/`.
6. `GrpcRequestRegistry` tracks in-flight requests so `handleGrpcRequestCancel`
   can run cleanup. Streaming handlers MUST register a cleanup fn there.

## Handler convention

- One file = one RPC. Export `async function <rpcName>(controller, request[, stream, id])`.
- Proto types come from `@shared/proto/dirac/<package>` (generated by `npm run protos`).
- Return proto `.create({})` instances (e.g. `Empty.create({})`), never raw literals.
- Unused controller/request → prefix `_` (`_controller`, `_request`).
- Subfolders are domains: `task/`, `state/`, `models/`, `file/`, `checkpoints/`,
  `browser/`, `worktree/`, `slash/`, `ui/`, `web/`, `stack/`, `commands/`.
- `commands/` = VS Code code-action entry points (`addToDirac`, `fixWithDirac`, …),
  legacy `dirac.*` id prefix kept for install compat — do not rename.
- `stack/` = local ailiance LLM stack control (start/stop, MCP allow/deny lists);
  pairs with `services/local-stack` + `services/local-router`.

## Subscriptions (`subscribeTo*`)

Long-lived streams pushing extension→webview events. Pattern: keep a module-level
`Set<StreamingResponseHandler>` of active subscribers, add on subscribe, register a
cleanup in the request registry, broadcast via a `send<Event>` helper exported for
the rest of the core to call (e.g. `sendStateUpdate`).

## Anti-Patterns

- Do NOT add handlers to a manual switch — register the RPC in `proto/`, regen, and
  drop the file in the matching domain folder. Routing is generated.
- Do NOT `import * as vscode` here — go through `HostProvider` (breaks standalone/CLI).
- Do NOT forget the registry cleanup on streaming subscriptions → memory leak +
  zombie streams after webview reload.
- Do NOT do heavy work synchronously in a handler before sending the first response;
  the webview blocks on `request_id` correlation.
- Do NOT mutate `controller.task` directly from a handler without going through the
  Task lifecycle helpers (`tryAcquireTaskLock*`) — concurrent RPCs race.
- Do NOT throw non-`Error` values; the handler wrapper stringifies `error.message`.
